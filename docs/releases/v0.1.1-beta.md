# zttp v0.1.1-beta

Curated release notes. Paste the sections below above the auto-generated
commit changelog when editing the GitHub Release. Mirrors `CHANGELOG.md`.

## Highlights

A focused follow-up to v0.1.0-beta that sharpens the expert authoring loop and
adds opt-in actor-style messaging. The agent now asks one clarifying question
when a request is genuinely ambiguous instead of guessing, and every expert
session records per-session metrics so you can see round-trips to first green
proof and the proven-path ratio of what you shipped. Handlers can also send,
receive, and redeliver messages between actors via the new
`zttp:queue` module.

## Added

- `zttp expert`: the agent asks one clarifying question when a request is
  materially ambiguous (which auth scheme, which route, what "safe" means) instead
  of guessing and spending its veto/round-trip budget on a misread.
- `zttp expert`: per-session metrics. Each session writes a `session_summary`
  event at close with turn count, model round-trips, verified-edit count,
  round-trips to first green proof, and a proven-path ratio. `/ledger` prints the
  live numbers mid-session.
- `zttp:queue`: opt-in actor-style messaging (`send`, `request`, `receive`,
  `ack`, `nack`, `reply`) backed by an in-memory `ActorQueue` with per-actor
  priority mailboxes, lease-based redelivery, and dead-lettering after max
  attempts. Enable with `--actor-queue`.
- `--workflow-queue`: opt-in persisted durable dispatch queue for
  `zttp:workflow` call/follow/fanout child requests. Child requests are
  leased while running and completed response parts are written before the
  parent durable step result is recorded, so an interrupted workflow can
  resume without re-dispatching a completed child. Requires durable storage
  and a system bundle; saga dispatch remains unsupported in queue mode.
