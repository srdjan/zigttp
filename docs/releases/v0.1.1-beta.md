# zigttp v0.1.1-beta

Curated release notes. Paste the sections below above the auto-generated
commit changelog when editing the GitHub Release. Mirrors `CHANGELOG.md`.

## Highlights

A focused follow-up to v0.1.0-beta that sharpens the expert authoring loop. The
agent now asks one clarifying question when a request is genuinely ambiguous
instead of guessing, and every expert session records per-session metrics so you
can see round-trips to first green proof and the proven-path ratio of what you
shipped.

## Added

- `zigttp expert`: the agent asks one clarifying question when a request is
  materially ambiguous (which auth scheme, which route, what "safe" means) instead
  of guessing and spending its veto/round-trip budget on a misread.
- `zigttp expert`: per-session metrics. Each session writes a `session_summary`
  event at close with turn count, model round-trips, verified-edit count,
  round-trips to first green proof, and a proven-path ratio. `/ledger` prints the
  live numbers mid-session.
