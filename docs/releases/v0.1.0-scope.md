# v0.1.0 Scope

What's in the first tag and what is deliberately out, so users evaluating the runtime can size it against their needs without reading the source.

## In Scope

**HTTP server.** Threaded HTTP/1.1 with bounded URL, query, header, and body limits; ETag-aware static files; canonical-path checks that defeat traversal and symlink escapes; per-request runtime isolation via `HandlerPool`.

**JS engine.** Two-pass compiler (parser to IR to bytecode), pre-execution bytecode verifier, NaN-boxed values, polymorphic inline caches with megamorphic recovery, tiered JIT (baseline plus loop-specialized optimized), generational GC with SIMD turbosweep, request-scoped arenas.

**JS subset.** ES5 plus arrows, template literals, destructuring, spread, `for...of`, optional chaining, nullish coalescing, `match`, `assert`, pipe operator, typed arrays. Disallowed surface (classes, async/await, regex, `null`, `==`, `var`, `while`, `switch`, `this`, `new`, `try/catch`) is parse-time rejected with suggestions. See [language-subset.md](../language-subset.md).

**Virtual modules.** 22 modules across `data/`, `http/`, `net/`, `platform/`, `security/`, `workflow/` directories (env, crypto, router, auth, validate, decode, cache, sql, service, fetch, websocket, io, durable, compose, scope, url, id, http, log, text, time, ratelimit). Each declares `required_capabilities`; sensitive SDK helpers and runtime callbacks enforce those declarations at call time.

**Compile-time analyzers.** Contract extraction with handler properties, type checker, flow checker with data labels (secret, credential, user_input), fault coverage, intent extractor, system linker. All I/O-free and wasm-safe so the analyzer compiles to wasm64 for the in-browser playground.

**Attestation.** Default-on Ed25519 JWS over `(bytecode_sha256, contract_sha256, policy_sha256)`, embedded in self-extracting binaries, served on `/.well-known/zigttp-attest`, verifiable by any third party with `zigttp verify <url>`. Identity at `~/.zigttp/attest/keypair.bin`.

**Local deploy.** `zigttp deploy` emits a self-contained binary plus a JSONL proof receipt. No network, no Docker, no credentials.

**Durable workflows.** `zigttp:durable` step recording, crash recovery, sleep/sleepUntil/waitSignal primitives backed by an on-disk oplog.

**Proof tooling.** Proof studio (HTTP), proof card TUI, proof ledger, ratchet command for spec monotonicity, contract diffing (`zigttp prove`), system linking (`zigttp link`).

## Deferred

**Hosted cloud deploy.** `zigttp deploy --cloud`, `login`, `logout`, `review`, `grants`, `revoke-grant` are gated at CLI dispatch and reject with a "not in this beta" message. Hosted control-plane, provider, registry, and OCI image orchestration code has been retired from the core repository; `zigttp deploy` is local-only for the beta.

**Evented I/O backend.** Zig 0.16's `std.Io.Evented` networking surface is incomplete (Dispatch and Uring return `error.NetworkDown`, Kqueue panics). The runtime ships a threaded-only HTTP server on every platform. Revisit when Zig stdlib networking matures.

**Multipart parser.** Handlers receive raw bodies; `multipart/form-data` is not parsed in the runtime. Handlers needing it must parse themselves, respecting quoted boundaries. Tracked for a future module.

**JIT code-cache eviction.** Compiled native code is allocated per `CompiledFunction` and freed on handler swap. There is no size-bounded LRU. Safe for short-lived FaaS processes; a future release will add eviction for long-running servers.

**Tier-2 JIT stability.** Optimized loop-specialized code paths exist (`packages/zigts/src/jit/optimized.zig`) and pass the test suite. Treat as opt-in for performance characterization, not as a stability guarantee at v0.1.0.

**WebSocket peer-sent close-code validation.** RFC 6455 frame parsing is correct; explicit validation of peer close codes is deferred. See [docs/internals/websocket-audit.md](../internals/websocket-audit.md).

## Supported platforms

- **macOS** (Darwin) on Apple Silicon and Intel.
- **Linux** (`ubuntu-latest` in CI) on x86_64 and arm64.
- **Windows**: not supported in v0.1.0.
- **Zig toolchain**: 0.16.0 stable. `build.zig.zon` declares `minimum_zig_version = "0.16.0"`. Pre-release nightlies are not supported.
